// automatic memory

Five lifecycle hooks fire on every session event. You never run a “save” command — Eagle Mem captures context automatically.

  • SessionStart — injects project overview + recent summaries
  • UserPromptSubmit — FTS5 search for relevant past memories
  • PostToolUse — records file touches with deduplication
  • Stop — extracts <eagle-summary>, strips <private> tags
  • SessionEnd — marks session as completed

// codebase awareness

Two commands give Eagle Mem instant knowledge of your project — no manual descriptions needed.

  • eagle-mem scan — detects languages, frameworks, structure, entry points, dependencies
  • eagle-mem index — chunks source files into FTS5-searchable blocks
  • Incremental — re-indexes only files that changed (mtime-based)
  • Auto-injected — scan results surface at every session start

// taskaware compact loop

Break complex multi-step work into subtasks that survive /compact. No more context loss on long tasks.

  • Plan — break work into subtasks stored in the DB
  • Execute — work on one task at a time
  • Compact — run /compact when context fills up
  • Resume — SessionStart re-injects memory + next pending task

// privacy & safety

Everything stays local. One SQLite file, no network calls, no external services.

  • <private> tags — stripped before any data reaches the database
  • WAL mode — concurrent session safety across 4-5 instances
  • Secret redaction — API keys and tokens scrubbed from observations
  • Per-project isolation — each project gets its own memory bucket
